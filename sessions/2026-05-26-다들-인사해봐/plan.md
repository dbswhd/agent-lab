# plan.md

## Goal
그룹 룸에 참여한 에이전트들이 서로 인사하고 각자의 역할을 명확히 소개하여, 이후 협업을 위한 기반을 마련한다.

## Scope
- 각 에이전트(Cursor, Codex, Claude)의 역할 및 강점 소개
- 협업 구조와 각자의 기여 방식 확인
- 향후 논의 주제 수렴 준비

## Non-goals
- 특정 기술 문제 해결 (아직 주제 미정)
- 코드 작성 또는 실행
- 에이전트 간 우선순위 결정

## Agent perspectives

**Cursor**
*(이번 라운드에서 응답 없음 — 역할 확인 필요)*

**Codex**
- 문제를 작은 단위로 분해하고 검증 가능한 가설 수립에 집중
- 현재 상태 vs. 목표를 분리하여 가장 빠른 검증 경로 제안
- 불확실한 부분은 명시적 가정과 확인 질문으로 분리
- 테스트 전략, 실패 가능성, 롤백 순서까지 정리하는 실행 중심 역할

**Claude**
- 맹점 찾기, 반증 가능성 검토, 범위 경계 확인 담당
- 독립적 시각 유지하며 놓치기 쉬운 부분을 비판적으로 짚는 역할
- 다른 에이전트와 협력하되 독자적 판단 유지

## Open questions
1. **Cursor**는 이 룸에서 어떤 역할을 맡는가? (응답 미확인)
2. 앞으로 다룰 실제 주제/문제는 무엇인가?
3. 에이전트 간 의견 충돌 시 조율 방식은?
4. Scribe(룸)의 개입 빈도와 중재 기준은?

## Recommended next actions
1. **Cursor 역할 확인** — 응답을 받아 세 에이전트의 역할 삼각형을 완성한다
2. **논의 주제 수렴** — 사용자로부터 실제 다루고 싶은 문제나 프로젝트를 제시받는다
3. **협업 규칙 간단히 합의** — 예: "Codex가 실행 계획 초안 → Claude가 리스크 검토 → Cursor가 구현 세부사항 보완"
4. **첫 번째 실질 태스크 시작** — 주제가 정해지면 즉시 plan.md를 해당 목표로 업데이트한다
