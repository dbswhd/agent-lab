# plan.md

## Goal
대화 참여자(인간 + 3개 AI 에이전트)가 공통 작업 기반을 확인하고, 첫 번째 구체적 목표를 정의하여 실행 가능한 상태로 진입한다.

## Scope
- 현재 세션: 역할 정렬 및 첫 목표 합의
- 대상 레포: `agent-lab` (모노레포 구조 가정)
- 참여자: 인간 1명, Claude, Codex, Cursor

## Non-goals
- 아직 특정 기능 구현 착수 (목표 미확정)
- 레포 전체 리팩터링
- 외부 시스템 연동 설계

## Agent perspectives

**Cursor**
구현 경로와 파일 단위 편집에 집중. 레포 구조(`src/`, `web/`, `workflows/`, `sessions/`)를 파이프라인으로 연결하는 것이 핵심. 추천 루프: workflow 정의 고정 → 로컬 실행 → `sessions/` 결과 확인 → UI/API 계약 동기화. 다음 편집 후보: 진입점 정리, `.env` 경로 정합, smoke path 확보.

**Codex**
실행 순서와 검증 기준 설계에 집중. 접근 순서: 목표 정의 → 제약 확인 → 핵심 가설 분리 → 빠른 검증 순서 결정 → 구현 → 테스트 → 리스크 정리. 위험도 높은 부분 우선 검증 원칙.

**Claude**
맹점 발견, 반증 가능성 검토, 범위 경계 설정에 집중. 현재 단계에서는 목표 자체가 아직 명시되지 않았음을 지적. 가정을 명시하고 검증 가능한 형태로 전환하는 역할.

## Open questions
1. **첫 목표는 무엇인가?** — workflow 실행 / web UI / multi-agent room 중 하나를 선택해야 함
2. 레포 실제 상태 확인 필요 — 가정된 구조(`src/`, `web/` 등)가 실제와 일치하는가?
3. 에이전트 간 역할 분담 경계 — 설계(Claude) / 순서(Codex) / 편집(Cursor) 구분이 실제 작업에서 충돌하지 않는가?
4. 세션 아티팩트(`sessions/`) 형식과 소유자는 누구인가?

## Recommended next actions
1. **인간 참여자가 첫 목표 하나를 선택** (workflow 실행 / web UI / multi-agent room)
2. Cursor가 레포 진입점과 실행 타겟을 탐색하여 현재 상태 보고
3. Codex가 선택된 목표를 5개 이하 검증 가능한 단위로 분해
4. Claude가 각 단위의 가정과 실패 조건을 명시
5. 위 결과를 바탕으로 `plan.md` 다음 버전 갱신
