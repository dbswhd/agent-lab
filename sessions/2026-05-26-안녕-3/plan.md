# plan.md

## Goal
대화 참여자(인간 + 3개 AI 에이전트)가 공통 작업 방향을 정의하고, 다음 실행 가능한 단계를 합의한다.

## Scope
- 현재 레포 구조(`src/`, `app/`, `web/`, `workflows/`, `docs/`, `sessions/`) 파악
- 로컬 실행, 멀티에이전트 룸 UX, 워크플로 엔진 중 우선순위 1개 선택
- 선택된 영역에 대한 최소 실행 가능한 작업 단위 정의

## Non-goals
- 아직 코드 수정 또는 도구 호출 없음 (어떤 에이전트도 실행하지 않음)
- 전체 레포 리팩터링 또는 아키텍처 전면 재설계
- 요구사항이 확정되기 전 구현 착수

## Agent perspectives

**Cursor**
레포 구조를 IDE 관점에서 분석. 진입점으로 `.env.example`, `Makefile`, `workflows/` 샘플을 제안. 작업 목표를 (A) 로컬 실행, (B) 멀티에이전트 룸 UX, (C) 워크플로 엔진 정리 중 하나로 좁힐 것을 요청.

**Codex**
문제를 작은 단위로 분해하는 역할 자임. `증상 확인 → 코드 경계 파악 → 재현 → 최소 수정 → 검증` 순서를 표준 흐름으로 제안. 재현 불가, 테스트 부재, 요구사항 모호성을 초기 리스크로 명시.

**Claude**
비판적 시각 담당. 현재 대화 목적이 불명확함을 지적. 맹점 발견 및 반증 가능성 검토에 집중할 것을 선언. 구체적 주제 확정을 선행 조건으로 요청.

## Open questions

2. 현재 막힌 지점(버그, 기능 부재, 문서 불일치 등)이 있는가?
3. `docs/01-CONTROLLED-WORKFLOW-SYSTEM`, `docs/04-multi-agent-room`이 현재 코드와 정합하는가?
4. `sessions/` 저장 포맷 및 로테이션 규칙이 정의되어 있는가?
