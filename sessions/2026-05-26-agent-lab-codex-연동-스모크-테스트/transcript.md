# Session transcript

**Topic:** Agent Lab Codex 연동 스모크 테스트


## Planner

- Agent Lab에서 Codex 호출을 시작했을 때 요청 본문과 지시사항이 누락 없이 전달되는가?
- Codex가 지정된 출력 형식(예: 마크다운 bullet list만 반환)을 정확히 준수하는가?
- 한글 주제 입력 시 Codex 응답의 인코딩, 가독성, 의미 보존에 문제가 없는가?
- Codex 응답이 Agent Lab 화면 또는 결과 저장소에 정상적으로 표시·기록되는가?
- 호출 실패, 타임아웃, 잘못된 응답 형식이 발생할 때 Agent Lab이 오류를 식별하고 재시도 또는 안내를 제공하는가?

## Critic

- 질문형 항목만 있고 성공 기준이 없다. 원문 보존, bullet-only 준수, 저장 성공을 판정할 구체적 기대값이 필요하다.
- 단일 정상 한글 입력만으로는 부족하다. 줄바꿈, 특수문자, 긴 프롬프트, 혼합 언어가 포함된 요청에서 누락·변형이 드러날 수 있다.
- “지시사항 전달”은 화면 응답만 보고 검증하기 어렵다. 전송 payload와 실제 저장 결과를 대조할 관측 지점이 빠져 있다.
- 형식 준수 실패가 Codex 문제인지 Agent Lab의 후처리·렌더링 문제인지 분리 검증이 없다.
- 저장소 기록 검증에는 중복 저장, 응답 잘림, 요청-응답 매핑 오류, 민감 지시사항의 과도한 로그 노출 여부도 포함돼야 한다.
- 재시도 검증은 실패 주입 없이는 입증되지 않는다. 타임아웃, 비정상 형식, 빈 응답, 부분 응답을 의도적으로 발생시켜야 한다.
- 동일 입력에 대해 payload, 원시 응답, UI 표시, 저장 데이터가 일치하지 않으면 이 스모크 테스트 계획은 통과 판정을 받을 수 없다.

## Scribe (plan)

## Goal
Agent Lab에서 Codex 연동의 기본 동작을 검증한다. 동일 요청에 대해 전송 payload, Codex 원시 응답, UI 표시, 저장 데이터가 일관되며, 오류 상황이 식별·처리되는지 확인한다.

## Scope
- 요청 전달 검증: 한글 주제와 출력 지시가 payload에 누락·변형 없이 포함되는지 확인한다.
- 출력 형식 검증: `markdown bullet list only` 같은 지정 형식에 대해 원시 응답과 화면 표시 모두 기대 형식을 준수해야 한다.
- 문자·내용 보존 검증: 한글, 줄바꿈, 특수문자, 혼합 언어, 긴 프롬프트 입력에서 인코딩 오류, 잘림, 의미 변형이 없어야 한다.
- 표시·저장 검증: 원시 응답, UI 표시, 저장 결과가 일치하고, 요청과 응답이 올바르게 매핑되며 중복 저장이 없어야 한다.
- 관측 지점 확보: payload, 원시 응답, 렌더링 결과, 저장 레코드를 대조할 수 있어야 한다.
- 오류 처리 검증: 타임아웃, 비정상 형식, 빈 응답, 부분 응답을 주입해 오류 식별, 사용자 안내, 재시도 동작을 확인한다.
- 로그 안전성 검증: 저장·로그에 민감 지시사항이나 불필요한 요청 전문이 과도하게 노출되지 않아야 한다.

## Non-goals
- Codex 응답 내용의 전문성 또는 품질 평가
- 대규모 부하, 장시간 안정성, 비용 최적화 검증
- Agent Lab 전체 기능 또는 타 모델 연동의 회귀 테스트
- 보안 감사나 개인정보 처리 정책 전반의 검토

## Open questions
- payload, 원시 응답, UI 표시, 저장 데이터를 각각 어디에서 관측할 수 있는가?
- 형식 위반을 Codex 원시 출력과 Agent Lab 후처리 문제로 어떻게 구분할 것인가?
- 저장 데이터의 허용 가능한 길이 제한과 로그 마스킹 기준은 무엇인가?
- 재시도 횟수, 타임아웃 기준, 사용자 안내 문구의 기대 동작은 무엇인가?
