# plan.md

## Goal
대화 참여자(Human + 3 AI 에이전트)가 공통 작업 맥락을 확인하고, 다음 실질적 작업 단계를 준비한다.

## Scope
- 현재 단계: 인사 및 방향 설정 (온보딩)
- 대상 저장소: `agent-lab` (초기 구조, 미커밋 상태)
- 주요 디렉터리: `app/`, `src/`, `web/`, `workflows/`, `docs/`
- 참여 에이전트: Cursor(IDE/코드 편집), Codex(분해·실행 계획), Claude(비판적 분석·범위 검토)

## Non-goals
- 아직 구체적 작업 요청이 없으므로 코드 실행·저장소 변경·도구 호출은 하지 않음
- 특정 기능 구현 또는 리팩터링 착수 (주제 확정 전)
- 에이전트 간 역할 고정 (작업 성격에 따라 유동적으로 조정)

## Agent perspectives

**Cursor**
초기 저장소 레이아웃(`app/`, `src/`, `web/`, `workflows/`, `docs/`)을 기준으로 편집 단위를 좁혀 가는 접근 선호. 우선 편집 후보: ① `README.md`/`docs/APP.md` 목적·실행법 정리 → ② `Makefile`/`pyproject.toml` 부트스트랩 고정 → ③ `web/` ↔ `src/` 연결 지점 단일화. "돌아가는 한 루프" 확보 후 역할 분담 제안.

**Codex**
목표·현재 상태·성공 조건을 먼저 분리한 뒤, 문제를 작은 단위로 나누고 실행 순서와 리스크를 정리하는 역할. 실행 명령·도구 사용은 명시적 요청 이후로 유보.

**Claude**
맹점 발견·반증 가능성 검토·범위 경계 설정에 집중. 현재 대화 목적이 불명확하므로 구체적 주제 확정이 선행되어야 비판적 분석이 가능하다고 판단.

## Open questions
1. Human이 이 세션에서 다루고자 하는 구체적인 주제 또는 문제는 무엇인가?
2. 저장소의 현재 실행 가능 상태는? (로컬 부트스트랩 가능 여부)
3. 우선순위: 문서화 → 최소 실행 루프 → UI 연결 순서가 맞는가?
4. 에이전트별 역할 경계를 지금 고정할 것인가, 작업 단위마다 유동적으로 할 것인가?

## Recommended next actions
1. **Human → 작업 주제 제시**: 만들거나 고칠 대상을 한 문장으로 명시
2. **Codex**: 주제 수신 후 목표·현재 상태·성공 조건 3분할 정리
3. **Cursor**: `docs/` 스펙 파일 기준으로 최소 runnable path 파일 목록 제안
4. **Claude**: 제안된 범위의 맹점·리스크 검토
5. 전체 합의 후 `plan.md` 갱신 및 첫 번째 편집 착수
